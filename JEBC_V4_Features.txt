========================================================================
JEBC V4 完全機能リスト＆最新マニュアル
Javascript Ecosystem Bypass To C
========================================================================

このツールは、WebAssemblyを使ったネイティブプログラムと、JavaScript/TypeScriptといったモダンなWeb技術を「1つのエコシステム」として統合し、最速でビルドするための専用コンパイラです。

V4への超特大アップデートにより、以下の「8つの神機能」を搭載しました！

------------------------------------------------------------------------
【1】C言語・Rust・Zig と JS/TS を1つのファイル（.jebc）に混在して書ける！（V4新機能✨）
------------------------------------------------------------------------
ファイルの拡張子を `.jebc` にするだけで、の中に C / Rust / Zig / JS / TS 全てを同居させられます。コンパイラが自動でそれぞれのネイティブ言語をwasmに変換し、1つのWASMバイナリに完璧にリンクさせます！

    lang.c {
        extern "C" { int calc_c() { return 10; } }
    }
    lang.rs {
        // Rustもそのまま書ける！
        #[no_mangle] pub extern "C" fn calc_rs() -> i32 { 20 }
    }
    lang.zig {
        // Zigもそのまま書ける！
        export fn calc_zig() i32 { return 30; }
    }

    lang.js {
        // JSから各言語の関数を直接叩ける！
        console.log(native.calc_c() + native.calc_rs() + native.calc_zig());
    }

------------------------------------------------------------------------
【2】「jebc dev」による神ローカル開発環境（V4新機能✨）
------------------------------------------------------------------------
Vite のように爆速の開発環境が手に入ります。
ターミナルで `jebc dev app.jebc` と入力すると、ローカルWebサーバーが立ち上がります。
そのままエディタでコードを書き換えて保存（Ctrl+S）すると、コンパイラが0.5秒未満で差分を検知し、C/Rust/JSを自動バックグラウンドリビルドします！

------------------------------------------------------------------------
【3】全自動・完全環境構築（V4新機能✨）
------------------------------------------------------------------------
`npm install -g jebc-install` を叩くだけで、以下のツール群が全て自動ダウンロード＆インストール＆自動パス通し（レジストリ登録）されます。
これでもう環境構築に一日溶かすことはありません！
・esbuild
・Emscripten (emsdk)
・Rust (rustup & wasm32-unknown-unknown)
・Zig

------------------------------------------------------------------------
【4】CSSとHTMLも直接埋め込めて完全な「フルスタック」に！
------------------------------------------------------------------------
JS/TS/C に加えて、CSSとHTMLのブロックもサポートしています。ビルド時に自動で `index.html` が生成されます。

    lang.html { <div id="app"></div> }
    lang.css { body { background: black; } }

------------------------------------------------------------------------
【5】C言語の構造体をTypeScriptのクラスに自動変換！（@jebc-sync）
------------------------------------------------------------------------
C言語側で定義した構造体に `// @jebc-sync` という目印をつけるだけで、自動で TS のクラス（ゲッター/セッター/メソッド完備）が生成されます！

------------------------------------------------------------------------
【6】JS/TSファイルの「完全なモジュール分離制」に対応！（V3機能）
------------------------------------------------------------------------
普通のJavaScriptと同じように、`import` と `export` が完全に使えます！
これにより「このファイルだけの変数」「この3つのファイルだけで共有する変数」を完璧に制御（カプセル化）できます。

    // --- file_a.jebc ---
    lang.ts {
        export const shared_data = 456; 
    }

    // --- file_b.jebc ---
    import { shared_data } from "./file_a.jebc";  // 別の.jebcファイルから自由に読み込める！
    lang.ts {
        console.log(shared_data);
    }

------------------------------------------------------------------------
【7】テキストまるごとコピペ合体機能「call」
------------------------------------------------------------------------
モジュールとして分離するのではなく、「単に別のファイルに書いたHTMLやCSS、C言語ヘッダをごっそり持ってきたい」という時は `call "components/ui.jebc"` が使えます。

------------------------------------------------------------------------
【8】面倒なことはすべて無視の「コマンド1発ビルド」
------------------------------------------------------------------------
Makefile も CMakeLists も、Webpack も一切必要ありません。
    `jebc main.jebc`
たったこれだけで、すべてが一瞬でビルドされます。
